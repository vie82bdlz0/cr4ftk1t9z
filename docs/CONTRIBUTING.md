# Contributing

Small site, but happy to take fixes.

## Getting started

- Fork, branch, edit, PR.
- Keep it vanilla. No bundlers, no frameworks.
- Stick to the existing CSS variables in `portfolio/style.css`.

## What is welcome

- Typo fixes.
- Accessibility improvements (focus states, color contrast).
- Performance: smaller images, fewer requests.
- Mobile breakpoints.

## What is not

- Frameworks or build steps.
- Unrelated dependencies.
- Sweeping rewrites without an issue first.

## Before opening a PR

- Check that the page still loads without JavaScript.
- Verify there is no console error in DevTools.
- Run an HTML validator if you touched markup.

Thanks for taking the time.
